Cómo funciona ahora

El nuevo flujo mantiene tu base y mejora la experiencia de registro.

1

El docente completa el formulario web

Todo ocurre dentro de APDocentePBA.

2

La web envía los datos al Apps Script

Los datos se procesan sin mostrar Google Forms.

3

Se guarda en `docentes`

Si el mail ya existe, la fila se actualiza.

4

Tu sistema APD sigue usando esa base

No hace falta crear otra hoja docente.

Preguntas frecuentes

Algunas dudas comunes sobre el nuevo registro integrado.

¿Se pierde el formulario anterior?

No. Simplemente ahora la web puede registrar directamente en la misma base.

¿Hace falta una hoja nueva?

No. Se usa la misma hoja `docentes` que ya tenías.

¿Qué pasa si el docente ya estaba registrado?

Se busca por correo y se actualiza su fila existente en lugar de duplicar datos.

¿Después se puede mejorar más?

Sí. El siguiente paso natural es permitir edición de preferencias y mostrar APD reales en la web.
